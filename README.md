# Procurement-sql-project

## 📌 Project Overview

This project focuses on analyzing supply chain procurement data using SQL.
The objective is to track purchase orders, supplier performance, procurement costs, delivery performance, inventory status, and transportation analysis.

## 🛠️ Tools Used

* SQL (MySQL)
* Database Management System
* Supply Chain Analytics Concepts

## 📂 Dataset Description

The project contains procurement transaction data with details such as:

* Purchase Order ID
* Order Date
* Supplier Name
* Material Details
* Category
* Quantity
* Unit Price
* Total Cost
* Delivery Date
* Expected Delivery Date
* Delivery Status
* Lead Time
* Location
* Transport Mode
* Inventory Status

## 🗄️ Database Structure

**Database Name:**

```
SupplyChain_Analytics
```

**Main Table:**

```
Procurement_Data
```

## 📊 Analysis Performed

### 1. Total Purchase Orders

Calculated the total number of purchase orders created.

### 2. Total Procurement Cost

Analyzed overall spending on procurement activities.

### 3. Supplier Performance Analysis

Measured supplier contribution based on:

* Number of orders
* Total purchase value

### 4. Delivery Performance

Calculated on-time delivery percentage to evaluate supplier reliability.

### 5. Delayed Delivery Tracking

Identified delayed purchase orders for improvement analysis.

### 6. Average Lead Time Analysis

Calculated average delivery time taken by suppliers.

### 7. Top Suppliers by Spending

Ranked suppliers based on total procurement cost.

### 8. Transportation Analysis

Analyzed orders based on transportation modes:

* Road
* Rail
* Air

### 9. Inventory Status Report

Reviewed inventory availability:

* Available
* Low Stock

### 10. Category Wise Purchase Analysis

Compared procurement spending across different material categories.

##  Business Insights

This analysis helps organizations to:

* Monitor supplier performance
* Reduce delivery delays
* Optimize procurement cost
* Improve inventory planning
* Support data-driven supply chain decisions
